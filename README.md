# 지식관리 — LLM과 함께 복리로 쌓는 회사 지식

경영자AI 밋업 **2차** 교안. 회사의 업무 파일을 생성·보관하는 단계부터 LLM 위키로 지식화하고, MCP를 통해 권한 범위 안에서 다시 꺼내 쓰기까지의 전 과정을 다룹니다.

## 교안 구성

| 섹션 | 내용 |
|---|---|
| 00 | 개요 · 학습목표 — 지식이 복리로 쌓이는 구조 |
| 01 | 준비하기 — 도구 10종 설치 (VS Code → Claude Code → 나머지는 AI에게) |
| 02 | 목표 시스템 다이어그램 — 8단계 순환 |
| 03 | Git의 운용 — 커밋·브랜치·템플릿·LFS·바이너리 관리·트리 시각화 |
| 04 | 업무환경의 데이터 생성 시스템 — 규칙강제 템플릿·hook·순찰자 |
| 05 | LLM 위키 구축 — 바이너리 컴파일 · graphify · Qdrant 의미 검색 |

| 07 | 정책 숙제 — 2차 합성지식의 분리 보관과 승격 담당자 |

## 이 교안의 원칙

- **AI로 관리한다** — 모든 실행 단계는 "LLM에게 이렇게 말하세요" 자연어 프롬프트가 기본이고, 그 프롬프트가 내부적으로 실행하는 CLI는 토글 안에 병기합니다. CLI에 능숙하면 토글의 명령을 직접 실행해도 됩니다.
- **VS Code 환경 전제** — 파일 구조는 탐색기, AI는 Claude Code 패널, 명령은 통합 터미널에서.
- **macOS · Windows 모두 지원** — 상단 스위치로 명령어가 한 번에 전환됩니다.
- **초보자 기준** — 각 단계마다 "왜 하는가 → 명령 → 예상 출력 → 성공 체크포인트 → 실수와 복구법" 순서로 구성했습니다.
- 상세 설명은 토글(`▸`) 안에 넣어 본문은 목차·명령 위주로 간결하게 유지했습니다.
